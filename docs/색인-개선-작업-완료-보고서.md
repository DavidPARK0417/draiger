### 2. Canonical URL 추가

모든 주요 페이지에 canonical URL 메타데이터를 추가하여 검색 엔진이 표준 URL을 명확히 인식하도록 개선했습니다.

#### 추가된 페이지:

**2.1. 인사이트 페이지 (`/insight`)**
- **파일**: `src/app/insight/layout.tsx`
- **Canonical URL**: `/insight`

**2.2. 검색 페이지 (`/search`)**
- **파일**: `src/app/search/layout.tsx` (신규 생성)
- **Canonical URL**: `/search`

**2.3. 후원 페이지 (`/support`)**
- **파일**: `src/app/support/layout.tsx`
- **Canonical URL**: `/support`

**2.4. 인사이트 상세 페이지 (`/insight/[slug]`)**
- **파일**: `src/app/insight/[slug]/page.tsx`
- **Canonical URL**: `/insight/{slug}`

**2.5. 메뉴 상세 페이지 (`/menu/[slug]`)**
- **파일**: `src/app/menu/[slug]/page.tsx`
- **Canonical URL**: `/menu/{slug}`

**효과**:
- 중복 콘텐츠 문제 방지
- 검색 엔진의 올바른 URL 인식 지원
- SEO 개선

### 3. BreadcrumbList 구조화된 데이터 추가

Google이 페이지 계층 구조를 더 잘 이해할 수 있도록 BreadcrumbList 스키마를 추가했습니다.

#### 3.1. 인사이트 상세 페이지

**파일**: `src/app/insight/[slug]/page.tsx`

**브레드크럼 구조**:
```
홈 > 인사이트 > [카테고리] > [글 제목]
```

**특징**:
- 카테고리가 있는 경우 4단계 구조
- 카테고리가 없는 경우 3단계 구조

#### 3.2. 메뉴 상세 페이지

**파일**: `src/app/menu/[slug]/page.tsx`

**브레드크럼 구조**:
```
홈 > 오늘의 메뉴 > [레시피 제목]
```

**효과**:
- 검색 결과에서 브레드크럼 표시 가능
- 사이트 구조의 명확한 전달
- 크롤러의 사이트 이해도 향상
- 내부 링크 구조 강화

## 📊 기술적 세부사항

### Canonical URL 구현

```typescript
export const metadata: Metadata = {
  title: "페이지 제목",
  description: "페이지 설명",
  alternates: {
    canonical: "/페이지-경로",
  },
  // ...
};
```

### BreadcrumbList 스키마 구현

```typescript
const breadcrumbData = {
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "홈",
      "item": baseUrl
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "인사이트",
      "item": `${baseUrl}/insight`
    },
    // ... 추가 항목
  ]
};
```

## 🎯 예상 효과

### 단기 효과 (1-2주)

1. **크롤링 효율 향상**
   - Canonical URL로 중복 페이지 문제 해결
   - BreadcrumbList로 페이지 간 관계 명확화

2. **색인 생성 속도 개선**
   - 구조화된 데이터로 크롤러의 페이지 이해도 향상
   - 사이트맵과 함께 작동하여 색인 생성 가속화

### 중기 효과 (1-2개월)

1. **색인된 페이지 수 증가**
   - 현재 5개 → 목표 50개 이상

2. **검색 결과 품질 향상**
   - 브레드크럼이 검색 결과에 표시될 가능성
   - 더 명확한 페이지 정보 제공

3. **SEO 순위 개선**
   - 구조화된 데이터로 검색 엔진 최적화
   - 내부 링크 구조 강화로 페이지 랭킹 개선

## 🔍 기존 SEO 개선 사항 (유지)

이미 구현되어 있던 SEO 최적화 기능들:

1. **사이트맵 최적화**
   - 재검증 주기: 30분 (1800초)
   - 자동 업데이트: ISR 방식

2. **페이지 재검증 주기**
   - 인사이트 상세 페이지: 30초
   - 메뉴 상세 페이지: 60초

3. **내부 링크 강화**
   - 인사이트 글 본문에 관련 메뉴 링크 삽입
   - 인사이트 글 하단에 관련 글 추천

4. **구조화된 데이터**
   - Article 스키마 (인사이트)
   - Recipe 스키마 (메뉴)
   - WebSite 스키마 (메인)
   - CollectionPage 스키마 (메인)
